# 要求内容

## 概要
OneTabからエクスポートしたYouTube動画URLのテキストデータを読み込み、YouTube Data API v3を使用して指定プレイリストへの追加・削除を行うCLIツール。

## ユーザーストーリー

- OneTabからエクスポートしたURLリストを `src/input/add.txt` に貼り付けて `make run` するだけでプレイリストに追加したい
- 削除したい動画を `src/input/remove.txt` に貼り付けて `make run` するだけでプレイリストから削除したい
- 実行時に十字キーで追加先・削除元プレイリストを選択したい（前回の選択がデフォルト）
- 過去に追加した動画を追跡し、再追加時に検知できるようにしたい

## 受け入れ条件

- `make run` で起動後、十字キーでプレイリストを選択できること
- 前回選択したプレイリストが次回のデフォルトとして表示されること
- 追加先と削除元で別々のプレイリストを指定できること（同一URLが両ファイルに存在してもOK）
- `playlist_history.csv` に全操作（追加・削除・対象プレイリストID）が記録されること
- `make auth` で初回OAuth2認証が完了し `token.json` が生成されること
- OneTabのURL形式（`youtube.com/watch`, `youtu.be`, `youtube.com/shorts`）を正しくパースできること
- 同一ファイル内の重複video_idは入力時点で除去されること
- Quota超過時はポーリングして再試行すること
- **処理の取りこぼしゼロ原則**（下記セクション参照）を遵守すること

## 処理の取りこぼしゼロ原則

ユーザーが意図した追加・削除が「知らない間にスキップされていた」という状態を防ぐ。
**すべての非正常ケースはユーザーに明示的に通知する。サイレントなスキップは禁止。**

### インタラクティブ確認（処理中に1件ずつ聞く）

| ケース | 挙動 |
|---|---|
| 追加対象がCSV履歴に存在する（同プレイリスト） | 「再追加しますか？[y/N]」と1件ずつ確認。Nならスキップ |

### 処理後レポート（完了後にまとめて表示）

以下のケースは処理中にも表示するが、**最後の完了サマリーにも全件一覧表示**する。

| カテゴリ | 内容 |
|---|---|
| 無効URL | video_idを抽出できなかった行（add.txt / remove.txt） |
| 重複スキップ | 同一ファイル内で重複していたvideo_id |
| プレイリスト既存（409） | APIが「すでに存在する」と返した動画（CSVとの不整合の可能性） |
| 削除対象が不在 | プレイリストに存在しなかった削除対象動画 |
| APIエラー | その他のAPIエラーで処理できなかった動画 |

### 完了サマリーの表示形式（イメージ）

```
── 完了 ─────────────────────────────────────────
✅ 追加成功: 3件 / 削除成功: 2件

⚠️  処理されなかった項目（要確認）

[無効URL - add.txt]
  ・ https://example.com/invalid | タイトル

[追加スキップ - すでにプレイリストに存在 (CSV未記録)]
  ・ dQw4w9WgXcY | 動画名

[削除スキップ - プレイリストに存在しなかった]
  ・ ABC123xxxxx | 動画名

[APIエラー]
  ・ XYZ789xxxxx | 動画名 → HTTP 500
─────────────────────────────────────────────────
```

⚠️ セクションがすべて空の場合は表示しない（全件正常処理）。

## 制約事項

- 認証方式: OAuth2（サービスアカウント不可・個人プレイリストのため）
- 実行環境: Docker コンテナ（docker-compose、polydock構成）
- inputファイルのクリアは行わない（ユーザーが手動管理）
- ローカルの `playlist_history.csv` を履歴の正本として扱い、API による全件取得は行わない
- APIリクエスト間に `INTERVAL_SECONDS`（デフォルト1.0秒）の待機を入れる

## 壁打ちで決定した事項

| 項目 | 決定内容 |
|---|---|
| 認証方式 | OAuth2（サービスアカウントはYouTube個人プレイリストに使用不可） |
| プレイリスト指定 | 実行時に十字キーで選択。追加と削除で別プレイリストを指定可能 |
| プレイリスト選択UI | `questionary.select()` で十字キー選択、前回選択をデフォルト表示 |
| 前回選択の保存 | `config.json` に `last_add_playlist_id` / `last_remove_playlist_id` を保存 |
| 重複除去 | `parse_input()` のパース時点でvideo_idの重複を除去 |
| inputファイルクリア | 不要（ユーザーが手動でペーストするため） |
| Quota対策 | リクエスト間に待機時間、超過時はポーリングで再試行 |
| CSV履歴スキーマ | `timestamp, video_id, title, url, action, playlist_id` |
| タイトル未記載 | タイトルなしとして記録（補完しない） |
| 準正常系の扱い | 409/404はスキップ正常扱い、その他エラーはスキップしてERRORログ |
