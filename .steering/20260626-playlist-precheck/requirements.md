# 要求定義: プレイリスト事前チェック機能

## 背景・課題

現在の `process_add` は、動画を追加する前にCSV履歴のみと照合している。そのため**他の手段でプレイリストに追加された動画**（ブラウザ、スマホアプリなど）を検出できず、重複追加が発生した。

## 変更内容

### 追加するチェックの順序

各動画を追加するとき、以下の順序で確認する：

```
① API事前チェック: プレイリストに既存？
    → YES: already_in_playlist に記録してスキップ（ユーザー確認なし）
    → NO:
        ② CSV事前チェック: 過去にこのツールで追加済み？
            → YES: ユーザーに1件ずつ確認（「追加しますか？」）
                → Yes: 追加実行
                → No: confirmed_skip にカウント
            → NO: そのまま追加実行
```

### シグネチャ変更

```python
# 変更前（history を利用していたが、was_previously_added チェックのみ）
process_add(youtube, playlist, history, add_file, csv_path) -> AddResult

# 変更後（シグネチャは同じ。ロジックを追加）
process_add(youtube, playlist, history, add_file, csv_path) -> AddResult
```

シグネチャは変わらない。内部ロジックを追加する。

### `AddResult.already_in_playlist` の意味変更

- 変更前: 409 Conflict（API が弾いた）
- 変更後: API事前チェックでプレイリスト既存と判明 → 自動スキップ

409 は事前チェックで防げるため、フォールバックとしては残すが主な経路ではなくなる。

## 制約事項

- `fetch_playlist_video_ids` のページネーションを実装し、50件超のプレイリストに対応する
- `fetch_playlist_video_ids` は `repository/youtube.py` に配置する（リポジトリ層の責務）
- `service/sync.py` は `fetch_playlist_video_ids` を呼び出すだけ（APIの詳細を知らない）
- 既存の75テストが引き続きすべてPASSすること

## ユーザーストーリー

> 追加先プレイリストを選んで `make run` を実行したとき：
> - プレイリストにすでにある動画は確認なしに自動スキップしてほしい
> - プレイリストにはないが、過去にこのツールで追加した履歴がある動画は「追加しますか？」と聞いてほしい
> - どちらにも当てはまらない動画はそのまま追加してほしい

## 受け入れ条件

- [ ] プレイリストに既存の動画は自動スキップされ `already_in_playlist` に記録される
- [ ] プレイリストになくCSV履歴にある動画はユーザー確認プロンプトが表示される
- [ ] ユーザーが「No」を選択した場合、`confirmed_skip` にカウントされる
- [ ] ユーザーが「Yes」を選択した場合、追加が実行される
- [ ] どちらにもない場合、確認なしに追加される
- [ ] 50件超のプレイリストでも全件チェックできる（ページネーション対応）
