# プロダクト要求定義書

## プロダクトビジョンと目的

**Footprints** は、OneTabからエクスポートしたYouTube動画URLをYouTubeプレイリストへ同期するCLIツールである。

ブラウザで「あとで見る」として溜めた動画を、手動コピーやWebUI操作なしに、テキストファイルへの貼り付けとコマンド一発で整理できる状態を目指す。

---

## ターゲットユーザーと課題・ニーズ

**ターゲット**: YouTubeを学習・情報収集に活用し、OneTabでタブを管理しているエンジニア・知識労働者

**課題**:
- OneTabに溜まったYouTube URLを手動でプレイリストへ移す作業が面倒
- ブラウザやスマホアプリで追加した動画と重複して追加されてしまう
- 過去に追加した動画を誤って再追加してしまう

**ニーズ**:
- URLをテキストファイルに貼り付けるだけで追加・削除を自動化したい
- 重複チェックをツールに任せたい
- どの動画が追加・削除されたかを履歴として残したい

---

## 主要な機能一覧

| 機能 | 説明 |
|---|---|
| 動画追加 | `add.txt` のURLをYouTubeプレイリストに追加する |
| 動画削除 | `remove.txt` のURLをYouTubeプレイリストから削除する |
| プレイリスト選択 | 実行時に十字キーでプレイリストを選択する（前回選択をデフォルト表示） |
| 重複チェック | APIによるプレイリスト既存確認・CSVによる過去追加確認の2段階で重複を防止する |
| 操作履歴管理 | 追加・削除操作を `playlist_history.csv` に記録する |
| Quota制御 | Quota超過時はポーリングで自動再試行する |
| OAuth2認証 | `make auth` で初回認証を完了し、以降は自動リフレッシュする |

---

## 成功の定義

- `make run` の実行だけで、`add.txt` / `remove.txt` の内容がプレイリストに反映される
- 処理後サマリーで成功・スキップ・エラーの全件が把握できる
- 意図しない重複追加が発生しない

---

## ユーザーストーリー・受け入れ条件

### US-01: 動画を追加する

> OneTabからエクスポートしたURLを `add.txt` に貼り付けて `make run` するだけでプレイリストに追加したい

**受け入れ条件**:
- `add.txt` に記載したURLがプレイリストに追加されること
- `watch?v=`, `youtu.be/`, `shorts/` など複数のURL形式に対応すること
- 同一ファイル内の重複URLは先勝ちで1件のみ処理されること
- プレイリストにすでに存在する動画は確認なしにスキップされること
- CSV履歴にある動画は「再追加しますか？」と確認してから処理されること

### US-02: 動画を削除する

> 削除したい動画を `remove.txt` に貼り付けて `make run` するだけでプレイリストから削除したい

**受け入れ条件**:
- `remove.txt` に記載したURLがプレイリストから削除されること
- プレイリストに存在しない動画はスキップされサマリーに表示されること

### US-03: プレイリストを選択する

> 実行時に十字キーで追加先・削除元プレイリストを選択したい

**受け入れ条件**:
- 実行時に自分のYouTubeプレイリスト一覧が表示されること
- 前回選択したプレイリストがデフォルトとして表示されること
- 追加先と削除元で別々のプレイリストを指定できること

### US-04: 処理結果を確認する

> 実行後に何が成功・スキップ・エラーになったか把握したい

**受け入れ条件**:
- 完了後に追加成功件数・削除成功件数が表示されること
- 無効URL・重複・APIエラー・スキップなど非正常ケースが全件サマリーに表示されること
- 全件正常処理の場合、警告セクションは表示されないこと

---

## 機能要件・非機能要件

### 機能要件

- OneTabの `URL | タイトル` 形式をパースできること（タイトルは省略可）
- 重複チェックは2段階（APIによるプレイリスト既存確認 → CSVによる履歴確認）で行うこと
- 履歴CSVは追加・削除ともに記録すること
- Quota超過（403 quotaExceeded）時は自動でポーリング再試行すること
- `--auth-only` フラグで認証のみ実行できること

### 非機能要件

- 実行環境: Dockerコンテナ（docker-compose）
- 認証方式: OAuth2（サービスアカウント不可・個人プレイリストのため）
- APIリクエスト間に `INTERVAL_SECONDS`（デフォルト1.0秒）の待機を入れること
- `playlist_history.csv` はGit管理外とし、外部ディレクトリにマウントすること
- `client_secret.json`, `token.json`, `config.json` はGit管理外とすること
